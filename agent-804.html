三、两个安全技能库的核心区别
▼
  • 框架差异：项目一 = PTES + 等保2.0 + ISO27001 + OWASP；项目二 = ATLAS + D3FEND + NIST AI RMF（AI原生）
  • 结构差异：项目一 = 39模块目录→模块内技能MD；项目二 = 762个独立目录→每个技能独立skill.md（YAML frontmatter标准化）
  • 语言差异：项目一 = 中文；项目二 = 英文
  • Star差异：项目一 = 未公开；项目二 = 4500+ Star（社区验证）
  • 合规建设：只有项目一有等保2.0 + ISO27001
  • DevSecOps：只有项目一有独立模块
  • AI Agent集成：项目二粒度更细（每个skill.md可直接被Agent解析）；项目一有CLI工具skill_query.py
  • 适用场景：项目一 = 渗透测试全流程 + 合规体系建设；项目二 = AI Agent自动化安全分析 + 威胁狩猎

四、如何根据场景选仓库
▼
  • 做渗透测试（授权项目） → 项目一，按PTES阶段走：信息搜集→漏洞扫描→漏洞利用→权限提升→后渗透→横向移动→持久化→清痕→报告
  • 建立合规体系（等保2.0/ISO27001） → 只在项目一，项目二无合规模块
  • 给AI Agent装备安全能力 → 项目二，每个skill.md含When to Use + Prerequisites + Workflow，可直接解析
  • LLM安全专项 → 项目一16号模块（中文，PTES延伸） + 项目二prompt-injection相关技能（英文，OWASP LLM Top10）
  • 威胁狩猎/恶意软件分析 → 项目二，762技能粒度细，ATLAS/D3FEND框架映射完整
  • 云安全渗透测试 → 项目二（48技能，AWS S3审计/Azure AD配置/云C2等）
  • 数字取证/应急响应 → 项目一（30-34号模块）+ 项目二（取证+应急独立技能）
  • DevSecOps安全集成 → 只在项目一（18号模块）
